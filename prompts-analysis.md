# Анализ расшифровок голосовых звонков

## Обзор

Этот промпт предназначен для комплексного анализа расшифровок телефонных разговоров между менеджерами и клиентами компании Ромашка. Анализ проводится по трем ключевым направлениям:

1. **Контроль кросс-сейла** - выявление упущенных возможностей предложения дополнительных услуг
2. **Знание продукта** - оценка компетентности менеджеров в продуктовой линейке
3. **Анализ успешных кейсов** - выявление примеров эффективного кросс-сейла

## Структура анализа

### 1. Контроль кросс-сейла

**Цель:** Найти моменты в диалоге, где менеджер мог бы предложить дополнительные услуги или продукты, но не сделал этого.

**Критерии анализа:**
- Клиент упоминает текущие потребности (например, "нужен сайт для продажи товаров")
- Менеджер предлагает только базовое решение, игнорируя смежные услуги
- В разговоре есть намеки на дополнительные потребности клиента
- Менеджер не спрашивает о смежных услугах или продуктах

**Примеры упущенных возможностей:**
- При обсуждении сайта не предложены услуги SEO, контекстной рекламы, аналитики
- При разговоре о ПО не упомянуты услуги технической поддержки, обучения, интеграции
- При обсуждении разработки не предложены услуги сопровождения и поддержки

**Шкала оценки:**
- **Высокий приоритет упущенной возможности**: Клиент явно выражает потребность в смежных услугах
- **Средний приоритет**: Клиент дает косвенные намеки на дополнительные потребности
- **Низкий приоритет**: Возможность кросс-сейла существует, но неочевидна

### 2. Знание продукта

**Цель:** Оценить, насколько хорошо менеджер ориентируется в продуктовой линейке компании.

**Критерии оценки:**
- **Отлично (5 баллов)**: Менеджер демонстрирует глубокое знание всех аспектов продукта, уверенно отвечает на вопросы, предлагает оптимальные решения
- **Хорошо (4 балла)**: Менеджер знает основные характеристики, но может допускать небольшие неточности
- **Удовлетворительно (3 балла)**: Менеджер знает базовую информацию, но испытывает затруднения с деталями
- **Плохо (2 балла)**: Менеджер дает неточную информацию или не может ответить на вопросы
- **Критично (1 балл)**: Менеджер демонстрирует полное незнание продукта

**Аспекты для оценки:**
- Точность предоставляемой информации о ценах, сроках, функционале
- Способность адаптировать предложения под потребности клиента
- Знание технических деталей и ограничений продуктов
- Умение сравнивать разные варианты решений

### 3. Анализ успешных кейсов

**Цель:** Выявить диалоги, где кросс-сейл был реализован успешно, для формирования библиотеки лучших практик.

**Критерии успешного кросс-сейла:**
- Менеджер активно предлагает дополнительные услуги
- Клиент проявляет интерес к предложению
- Достигнута договоренность о дополнительных услугах
- Менеджер использует эффективные техники продаж

**Элементы анализа:**
- **Триггеры кросс-сейла**: Что побудило менеджера предложить дополнительную услугу
- **Техники убеждения**: Какие аргументы использовались
- **Реакция клиента**: Как клиент отреагировал на предложение
- **Результат**: Была ли достигнута договоренность

## Процесс анализа

### Шаг 1: Предварительный анализ файла
```
Прочитать YAML frontmatter для получения контекста:
- Тип звонка (direction: inbound/outbound)
- Длительность разговора
- Имя менеджера
- Категория звонка (продажи/поддержка/претензия)
```

### Шаг 2: Структурирование диалога
```
Разделить разговор на логические блоки:
- Приветствие и установление контакта
- Выяснение потребностей клиента
- Предоставление основной информации
- Заключительная часть и договоренности
```

### Шаг 3: Поиск паттернов кросс-сейла
```
Проанализировать каждый блок на предмет:
- Упущенные возможности (что можно было предложить дополнительно)
- Успешные примеры кросс-сейла
- Знание продукта менеджером
```

### Шаг 4: Формирование выводов
```
Для каждого типа анализа предоставить:
- Конкретные цитаты из диалога
- Обоснование оценки
- Рекомендации по улучшению
```

## Инструменты анализа

### Автоматизированные проверки:
1. **Поиск ключевых слов** для выявления упущенных возможностей
2. **Анализ временных меток** для понимания структуры разговора
3. **Сравнение с эталонными скриптами** продаж

### Ручные проверки:
1. **Контекстный анализ** каждого упоминания продуктов/услуг
2. **Оценка эмоционального состояния** участников разговора
3. **Проверка логичности** рекомендаций менеджера

## Лучшие практики на основе анализа

### Для менеджеров:
- **Активное слушание:** Задавать уточняющие вопросы для выявления скрытых потребностей
- **Комплексные предложения:** Всегда предлагать пакет решений
- **Подтверждение понимания:** Перефразировать потребности клиента

### Для обучения:
- **Создание библиотеки кейсов:** Сбор успешных примеров кросс-сейла
- **Регулярные тренинги:** Обучение техникам выявления потребностей
- **Мониторинг прогресса:** Отслеживание улучшения показателей

## Примеры анализа

### Пример 1: Упущенная возможность кросс-сейла
```
Клиент: "У нас небольшая торговая компания. Нужен сайт для продажи товаров онлайн."

Менеджер: "Отлично. Для интернет-магазина рекомендую начать с базовой версии..."

Упущено: Не предложены услуги SEO, рекламы, интеграции с 1С
```

### Пример 2: Успешный кросс-сейл
```
Менеджер: "Помимо сайта, рекомендую также настроить контекстную рекламу в Яндекс.Директ"
Клиент: "Да, это тоже нужно"
Менеджер: "Отлично, тогда можем включить в пакет"

Успех: Активное предложение + положительная реакция клиента
```

## Метрики эффективности

### KPI для отслеживания:
- **Среднее количество упущенных возможностей** на звонок
- **Процент звонков с успешным кросс-сейлом**
- **Средняя оценка знания продукта** по шкале 1-5
- **Конверсия рекомендаций** в дополнительные продажи

### Целевые показатели:
- Упущенные возможности: < 2 на звонок
- Успешный кросс-сейл: > 30% звонков
- Знание продукта: > 4.0 баллов средняя оценка
