# Правила работы с голосовыми вызовами (Voice Calls Rules)

## Обзор

В рамках AI-First Workspace Template голосовые вызовы представляют собой структурированные записи коммуникаций с клиентами, партнерами и коллегами. Эти записи служат основой для анализа, автоматизации процессов продаж, маркетинга и операционной деятельности.

## Структура файлов

### Формат именования файлов
```
{YYYY-MM-DD}_{HH-MM-SS}_{phone_number}_{call_id}.md
```

**Примеры:**
- `2026-01-25_18-44-00_+79375553322_MTIzNDU2.md`
- `2026-01-25_18-50-00_+79375554433_MTIzNDU3.md`

### Обязательные компоненты:
- **Дата и время начала**: `YYYY-MM-DD_HH-MM-SS`
- **Номер телефона**: В международном формате `+7XXXXXXXXXX`
- **Call ID**: Уникальный идентификатор звонка (base64 или UUID)
- **Расширение**: `.md` (Markdown)

## Формат содержимого

### YAML Frontmatter (обязательно)
```yaml
---
call_id: "MTIzNDU2"              # Уникальный ID звонка
date_start: "2026-01-25 18:44:00" # Дата и время начала
date_end: "2026-01-25 18:49:00"   # Дата и время окончания
duration: 312                     # Продолжительность в секундах
from_number: "+79375553322"       # Номер звонящего
to_number: "+74950000000"         # Номер принимающего
direction: "inbound"              # inbound/outbound
manager: "Иванов Иван"            # Имя менеджера (если доступно)
---
```

### Секция расшифровки (обязательно)
```markdown
# Расшифровка звонка

**[00:00] Менеджер:** Добрый день, компания Ромашка.
**[00:05] Клиент:** Здравствуйте, я хотел бы узнать цену...
```

## AI-First принципы работы

### 1. Автоматизированная обработка
- **Автоматическая транскрибация**: Все входящие/исходящие звонки автоматически конвертируются в текст
- **Метаданные**: Автоматическое извлечение технических параметров звонка
- **Классификация**: Автоматическое определение типа звонка (продажи, поддержка, претензии)

### 2. Структурированный анализ
- **Извлечение интентов**: Автоматическое определение целей клиента
- **Sentiment analysis**: Анализ эмоционального состояния участников
- **Ключевые моменты**: Автоматическое выделение важных фраз и решений

### 3. Интеграция с бизнес-процессами
- **CRM интеграция**: Автоматическая синхронизация с системами управления клиентами
- **Задачи и действия**: Автоматическое создание follow-up задач
- **Отчетность**: Генерация аналитики по звонкам для менеджмента

## Аналитические возможности

### Метрики для отслеживания:
- **Средняя продолжительность** звонков по категориям
- **Конверсия** входящих звонков в сделки
- **Качество обслуживания** (sentiment score)
- **Эффективность менеджеров** (продажи/звонок)
- **Частые вопросы** и темы обсуждений

### Автоматизированные отчеты:
- Ежедневные/еженедельные сводки по звонкам
- Анализ трендов и сезонности
- Рекомендации по улучшению процессов

## Технические требования

### API интеграции
- **Mango Office**: Для получения метаданных звонков
- **Speech-to-Text**: Google Speech API или аналог для транскрибации
- **CRM системы**: Автоматическая синхронизация результатов

### Форматы данных
- **Аудио**: MP3/WAV с метками времени
- **Текст**: Markdown с временными метками [MM:SS]
- **Метаданные**: YAML frontmatter для структурированного хранения

## Правила создания контента

### Для менеджеров:
1. **Всегда использовать** временные метки в формате [MM:SS]
2. **Указывать роли**: Менеджер/Клиент/Партнер/Коллега
3. **Фиксировать решения**: Явно выделять достигнутые договоренности
4. **Отмечать эмоции**: [недоволен], [рад], [спокоен] при необходимости

### Для AI-ассистентов:
1. **Автоматически генерировать** структуру файла при новом звонке
2. **Заполнять метаданные** из API систем телефонии
3. **Создавать расшифровку** с точными временными метками
4. **Добавлять теги** для категоризации (#продажи, #поддержка, #претензия)

## Рабочие процессы

### 1. Входящий звонок
```
1. Звонок поступает → API Mango фиксирует метаданные
2. AI начинает транскрибацию в реальном времени
3. Создается файл с базовой структурой
4. После завершения → финализация файла
5. Автоматический анализ и интеграция с CRM
```

### 2. Исходящий звонок
```
1. Менеджер инициирует звонок через систему
2. AI мониторит и транскрибирует разговор
3. Создается файл с метаданными
4. После завершения → анализ эффективности
5. Обновление CRM и создание follow-up задач
```

### 3. Анализ и отчетность
```
1. Ежедневный парсинг новых файлов
2. Агрегация метрик по категориям
3. Генерация инсайтов и рекомендаций
4. Автоматическое создание отчетов для менеджмента
```

## Безопасность и приватность

### Правила хранения:
- **Шифрование**: Все аудиофайлы шифруются
- **Доступ**: Ограничение по ролям (менеджер видит только свои звонки)
- **Удаление**: Автоматическое удаление через 2 года (согласно требованиям)
- **Логирование**: Аудит всех доступов к записям

### Соблюдение законов:
- **179-ФЗ**: Согласие на запись разговоров
- **152-ФЗ**: Защита персональных данных
- **Документирование**: Фиксация всех согласий в CRM
